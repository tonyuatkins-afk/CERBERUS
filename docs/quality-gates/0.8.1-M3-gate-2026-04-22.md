## WARN items

### W14 (WARN, carried from M1): DGROUP AT RISK

Headroom vs 62 KB soft target fell from M1's 2,016 bytes to 1,824 bytes. M3 added only +192 bytes DGROUP (under the 350-byte cap from the plan); the AT RISK status is accumulated pressure, not an M3 overrun.

Before M2 (cache depth) lands, this gate recommends the same CONST-segment audit that W14 flagged in M1. M2's cap is 500 bytes; if that lands cleanly we finish 0.8.1 at ~1,300 bytes vs soft target — still in the yellow zone but not dropping further than M3 already did.

### W15 (WARN, new): M3.1 IIT discrimination is a stub pending real-iron signature

The `fpu_probe_iit_3c87()` function currently returns 0 unconditionally. The DB row and routing are in place so that a follow-up single-function edit will activate IIT detection, but the current binary behaves identically to v0.7.1 / v0.8.0 on 386-era systems (tags as generic "387"). The M3.1 exit criterion is explicitly hardware-gated.

Recommend: include the 386 DX-40 + IIT 3C87 capture in the next real-iron session. Capture needed: FNSAVE-buffer dump at offsets 0..107 after an FNINIT + FNOP sequence, so we can observe which bytes differ from Intel 387 and pick the discrimination signature empirically.

### W16 (WARN, new): M3.2 Genoa ET4000 probe is documented but unvalidated

The 3CDh read-write-readback probe is sourced from Tseng ET4000 programmer's reference documentation but has not been verified against the actual Genoa ET4000 board on the 386 DX-40 bench box. If the OEM's implementation of the ET4000 silicon differs in how 3CDh preserves written values, the probe may either misfire (false positive on a non-ET4000 card) or miss (false negative on the Genoa board).

Recommend: observe the 386 DX-40 capture's emitted keys. If `video.chipset=ET4000` lands on the Genoa board, the probe is validated. If it doesn't, we either refine the probe (require both patterns to round-trip identically) or fall back to the generic VGA path.
