## WARN items

### W9 (WARN): CGA snow gate uses per-character polling loop

`tui_wait_cga_retrace_edge()` polls 3DAh bit 0 twice per VRAM write. On CGA that's ~10-20 microseconds of spin per character. Acceptable cost for snow-free rendering but not free. The loop has a 64K-iteration timeout so a dead status port (broken emulator) cannot hang the UI forever; the timeout expires and writes proceed with possible snow.

Recommend: accept for M3. If real-iron CGA captures show the UI too slow to be comfortable, a future pass could gate per-burst (one retrace per 80-char row) rather than per-character. Not required.

### W10 (WARN): Adapter-tier refinement did not add Hercules-variant discrimination

The MS-DOS UI-UX research names three Hercules variants (HGC, HGC+, InColor) distinguishable via 3BAh bits 6:4. CERBERUS emits only `ADAPTER_HERCULES` generic. Adding variant-specific types would require adapter_t enum expansion and downstream changes. Out of M3 scope ("refine the waterfall", not "expand adapter coverage").

Recommend: defer to M4 or later. Current behavior — anyone on HGC/HGC+/InColor gets the same baseline Hercules rendering — is correct.

### W11 (WARN): DGROUP headroom after M3 is 2,608 bytes — tight

Within the 2 KB hard-reserve window, 608 bytes above it. Any significant M4 docs-parity additions (help text expansion, consistency rule narration, etc.) should run `wmake dgroup-report` before committing.

Recommend: M4 checks DGROUP before each doc addition. If headroom drops below 2 KB hard reserve, consider const-segment relocation via Watcom `-zc` (with the caveat that v0.6.1's attempt showed it doesn't move unnamed string literals, only explicitly const-qualified declarations).
