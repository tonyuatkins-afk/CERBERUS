# CERBERUS v0.8.0, Trust-First Release (draft)

**Draft release notes.** Tag pending once M4.8 quality gate closes the milestone set. Covers the 2026-04-21 through 2026-04-22 arc (milestones M1, M2, M3, M4) and closes the v0.7.1 real-iron gap that surfaced when BEK-V409 reproduced none of the DOSBox-X stability profile.

## Headline

v0.8.0 is the first CERBERUS release where every shipped result has been verified on real iron. The cut is smaller than the v0.7.1 envelope on purpose: features that could not be trust-proved on a 486 DX-2-66 inside the milestone window were either compiled out (upload), emit-suppressed (Whetstone), or deferred to 0.8.1 (IEEE-754 edge cases, CSV output). The result is a build that ships fewer numbers but every number it ships has a real-iron signature under it.

## What shipped

### M1. Trust-first foundation (2026-04-21)

- **Whetstone emit suppressed** (M1.1). Kernel still compiles, host tests still run against it, but the dispatcher emits `bench.fpu.whetstone_status=disabled_for_release` instead of a K-Whet number. Reason: the Curnow-Wichmann kernel has never produced a value in the published reference band on real 486 silicon (measures 10-30x low), and a trust-first release cannot ship an out-of-range number regardless of confidence label. Re-enable via `wmake WHETSTONE=1` for research builds. (`src/bench/bench_whetstone.c`, `docs/methodology.md` "Why Whetstone is not in 0.8.0")
- **Upload compiled out of stock builds** (M1.2, M1.3). `#ifdef CERBERUS_UPLOAD_ENABLED` gates every HTGET-era call site; the stock stub returns `UPLOAD_DISABLED` and sets `upload.status=not_built`. `/UPLOAD` on the CLI rejects at parse time. The barelybooting.com upload was the v0.7.1 stack-overflow culprit when the server was unreachable; compiling it out removes the attack surface entirely for 0.8.0. Issue #9 nickname buffer leak fixed at the same time via `_nmalloc` with static fallback. (`src/upload/upload.c`, `src/main.c`)
- **cpu.class normalized to family tokens** (M1.4). CPUID-class CPUs now emit `cpu.class=486` / `pentium` / `pentium_pro` / `pentium_4` instead of the vendor string. Vendor string remains available as `cpu.vendor`. Unblocks consistency engine rules that gate on CPU class. (`src/detect/cpu.c`)
- **bench_cpu DB anchor widened** (M1.5). 486 DX-2-66 iters_low lowered from 4,700,000 to 1,500,000 after BEK-V409 real-iron measurement of ~1.96M iters/sec. The original band was seeded from DOSBox-X-on-modern-host measurement and was too narrow. (`hw_db/cpus.csv`)
- **DGROUP audit tooling** (M1.6). `tools/dgroup_check.py` parses `cerberus.map` and reports near-data usage per segment. `wmake dgroup-report` surfaces it. Current near-data footprint: 60,976 bytes (target 62,000 soft ceiling; hard ceiling 65,536).
- **End-of-run hang fixed via `_exit()` bypass** (M1.7). BEK-V409 was hanging in Watcom libc teardown (atexit / FPU / stdio close) after Q-exit. Fix: replace `return exit_code` with `crumb_exit(); _exit((int)exit_code);` so INT 21h AH=4Ch fires directly. Safe because CERBERUS registers no atexit handlers and closes every FD explicitly before exit. Full investigation in `docs/methodology.md` "Final-exit `_exit` bypass". (`src/main.c`)
- **Three-tier real-hardware validation corpus** (M1.8 / M1.9). Captures archived in `tests/captures/`:
  - 486 tier: BEK-V409 (Intel i486DX-2-66, AMI 11/11/92, S3 Trio64, Vibra 16S) — `486-BEK-V409-2026-04-21-m1-exit/`
  - 386 tier: companion 386 DX-40 + IIT 3C87 + Aztec ISA audio + Genoa ET4000 — `386-real-2026-04-21-m1/`
  - 286/8088 tier: README claims retraction until a live 286 or 8088 capture lands; header claim text matched to supported tiers.

### M2. FPU behavioral fingerprint + cache stride extension (2026-04-22)

- **Cache stride=128 added** (M2.1, research gap C). Six-stride sweep (32/64/128/256/512/1024) now supports both `line_bytes=32` (Pentium-era) and `line_bytes=64` (Pentium-Pro+) inference via plateau detection. The previous 5-point sweep could not distinguish the two. Backward-compatible with 5-point captures via `inferred_from_5point_sweep=1` tag. (`src/bench/bench_cache_char.c`)
- **FPU FPTAN probe** (M2.2, research gap I). 5th axis of the behavioral fingerprint. 387+ pushes 1.0 after tan(angle); 8087/287 leave only the tangent. `diagnose.fpu.fptan_pushes_one=yes|no`.
- **FPU rounding-control cross-check** (M2.3, research gap J). All 4 rounding modes (nearest-even, down, up, truncate) against FISTP(1.5) and FISTP(-1.5), tabulated against canonical 387-class result table. `diagnose.fpu.rounding_modes_ok=yes|no` with 8 sub-keys for raw results.
- **FPU precision-control cross-check** (M2.4, research gap K). 3 precision modes (24/53/64-bit significand) against 1.0/3.0 stored as 10-byte tword; results must all differ. `diagnose.fpu.precision_modes_ok=yes|no`.
- **FPU exception-flag roundtrip** (M2.6, research gap M). Each of the 6 x87 exceptions (IE/DE/ZE/OE/UE/PE) triggered, observed in status word, cleared via FCLEX. `diagnose.fpu.exception_flags_ok=yes|no`.
- **Memory checkerboard + inverse checkerboard patterns** (M2.7). 0x55/0xAA and 0xAA/0x55 patterns added to `diag_mem` for adjacent-cell coupling fault coverage.

### M3. CUA-lite UI + MS-DOS UI/UX research Tier 0 alignment (2026-04-22)

- **CGA snow safety gate** (M3.1). `tui_wait_cga_retrace_edge()` polls 3DAh bit 0 for a retrace edge before any row-24 write or full-screen update on CGA. No-op on MDA/EGA/VGA. Cost ~1 ms per call on CGA, invisible at legend-refresh cadence. (`src/core/tui_util.c`)
- **F-key legend replaces row-24 status bar** (M3.2). Norton-style legend `F1 Help | F3 Exit | Esc Cancel` in Borland TStatusLine palette (0x30 base, 0x3F hotkey).
- **F1 help overlay** (M3.3). Static content overlay summarising CUA keys, verdict semantics, and how to read the three-pane UI. Dismiss via any key.
- **F3 exit + CUA key grammar formalized** (M3.4). F3 quits, Esc cancels modal overlays and scrollable views, Q remains a legacy exit alias, Enter and arrow keys navigate. Grammar documented in `src/core/ui.c` NAV_* enum and asserted by `tests/host/test_ui_nav.c`.
- **/MONO flag with semantic role mapping** (M3.5). Five-role palette (NORMAL/EMPHASIS/DIM/HIGHLIGHT/WARNING) maps to MDA-appropriate attributes (0x07, 0x0F, 0x01, 0x70, 0xF0). Forces mono rendering regardless of detected adapter tier. (`src/core/display.c`)
- **AX=1003h BL=00h 16-background-color enable** (M3.6). Issued at `display_init()` on EGA/VGA so WARNING's 0xF0 renders as bright background rather than blinking. No-op on MDA/CGA where the blink bit is immutable.
- **Adapter-tier waterfall refined** (M3.7). Four-probe cascade: INT 10h AH=1Ah (VGA BIOS) → INT 10h AH=12h BL=10h (EGA BIOS) → BDA 40:49h + 40:10h → 3BAh toggle (MDA vs CGA). Result cached for the run.

## What is deferred

### Deferred to 0.8.1

- **M2.5 IEEE-754 edge-case diagnostic** (research gap L). 9 operand classes x 5 ops. Deferred because of DGROUP budget pressure after M2.2/2.3/2.4/2.6 landed.
- **M2.8 CSV output mode** (`/CSV` flag, research gap H). Scope-cut to keep M2 within the milestone window.
- **L1 pointer-chase latency probe** (research gap A).
- **L2 cache detection via 64/128/256 KB sweeps** (research gap B). Needs DGROUP reclaim + FAR buffer work beyond the current 32 KB cap.
- **DRAM latency derivation** (research gap E).
- **BEK-V409-specific BSS overwrite root-cause investigation** (W4+W6 from M1 gate). Empirically localized to 3 probe paths (S3 Trio64, Vibra 16S/OPL fallback, UMC491 PIT). Removal-at-a-time protocol documented; needs a focused BEK-V409 session with KERNEL-mode debugger access.
- **IIT 3C87 FPU discrimination**, Genoa ET4000 detection, Hercules variant discrimination.

### Deferred to 0.9.0

- **Per-instruction FPU microbenchmarks** (research gap N). Replacement story for Whetstone: measure FADD / FMUL / FDIV / FSQRT / FSIN / FCOS cycle counts directly.
- **Upload path re-enabled** once barelybooting.com server-side is live and a stack-overflow-proof offline-fallback is shipped.
- **8088/XT-tier real-hardware validation** — README header claim matches supported tiers until a live 8088 capture lands.
- **Disk throughput benchmarks** via INT 13h raw sector reads.

## Real-iron captures that earned the ship

- **BEK-V409 clean exit capture** (2026-04-21 20:39). After M1.7 `_exit()` fix, pressing Q after UI summary returns cleanly to the `C:\CERBERUS>` prompt with no reset required. First time in the v0.7.x→v0.8.0 arc this has been reproducible on real 486 silicon. Capture: `tests/captures/486-BEK-V409-2026-04-21-m1-exit/capture.ini`.
- **386 DX-40 + IIT 3C87 capture** (2026-04-21). Companion capture on the 386 bench box confirms no BEK-V409-specific BSS overwrite on that tier (no `*** NULL assignment detected` canary), confirms Watcom libc teardown runs cleanly there, confirms adapter-tier waterfall identifies Genoa ET4000 as SVGA (chipset-specific discrimination deferred). Capture: `tests/captures/386-real-2026-04-21-m1/capture.ini`.
